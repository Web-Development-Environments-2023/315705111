    Q&A

    1. Differences:
    1) In Web1.0 we only had a static web where users could only consume information that was posted by people with the resources to host websites, while in Web2.0 we had a dynamic web where users could both upload and download information and participate in communities that created content (ex. YouTube), and now we have Web3.0 which is a semantic web where computers use ML and AI to understand the meaning of data and tie it to other data in an intelligent way.
    2) Web1.0 was built for static data presentation using mainly HTML, CSS and JavaScript. Web2.0 began utilizing frameworks for more dynamic and interactive web applications, suck as AJAX, Ruby on Rails, and PHP and also started focusing on providing APIs for better connectivity. Web3.0 focuses on even more advanced technologies like AI and ML for complex and personalized interaction, and we saw the usage of IoT and Blockchain to decentralize, autonomize and interconnect disparete parts of the web for a move away from big companies controlling the web.
    3) Web1.0 sites were mostly just text and images with simple animation, Web2.0 saw the integration of embedded videos, 2D portals and more robust website design, and Web3.0 has an enormous range of option for presentation, from integrated games in the browser to Multi-user virtual environments, to 3D applications that run in the webpage.

    2. Comparison:
    Use-Case: Searching for causes of stomach-ache.
    Web1.0: You are presented with a static site that will display text relating to "things that cause stomach pain", most likely information published by a large company.
    Web3.0: You are presented with an aggregation of data from multiple sources which is personalized according to your location, your age and sex, your previous search history and any other relevant information that has been fed into the search algorithm automatically. The page can contain more that just relevant information, it can contain links to online stores that sell painkillers or antacids, it can provide the link to contact your family doctor(lord knows how it got that information...) and it can extrapolate from your previous searches for fast food resteraunts and provide ads for gym memberships.

    This Use-Case covers all 3 Differences: static -> personalized, HTML CSS and JS -> PHP and other robust frameworks needed for implementation, and text -> robust animated interactable marvels of technology.
